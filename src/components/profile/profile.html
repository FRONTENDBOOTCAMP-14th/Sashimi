❗️ 해당 작업 영역의 시행착오

  • ❓ 문제점

    HTML과 CSS만으로 구현해야 했기 때문에 버튼을 눌렀을 때, 어떤 요소의 상태 변화 처리가 어려웠다. 원래 어떠한 기능을 가지고 있을까를 고려해서 프로필 편집/완료 버튼에 <button> 태그를 사용했었다. 페이지 이동처럼 보이도록 <form action="/index.html">으로 감싸고 <button type="submit">을 사용해 임시로 구현했었다. 그러나, ‘프로필 편집’ 버튼은 실제로 단순한 페이지 이동만을 위한 요소였기 때문에 submit 타입을 사용하는 것이 적절하지 않았다는 점을 인지하게 되었다.

  • 💡 해결방식

    페이지 이동이 주목적이라면 <a>태그가 더 적절하다고도 판단했지만, 두 버튼을 동일한 컴포넌트로 관리해야 하므로 한쪽만 <a>로 바꾸면 컴포넌트 재사용성이 떨어진다고 생각했다. 결국 두 버튼 모두 <button type="button">으로 통일하고, JavaScript로 페이지 이동 동작을 제어하는 방식으로 개선했다. 이로써 시맨틱 마크업을 유지하면서도 컴포넌트의 일관성과 재사용성을 확보할 수 있었다.
