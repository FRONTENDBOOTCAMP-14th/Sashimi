❗️ 해당 작업 영역의 시행착오

  • ❓ 문제점

    VOD 목록들의 타이틀 텍스트 옆에 붙는 빨간색의 작은 점을 타이틀 텍스트 영역의 가상 요소로 생성하여 처리하였고 포지션 값을 주어 위치를 배정하는 간단한 작업이였다. 따로 width 값을 주지 않았기에 텍스트의 길이만큼 늘어난 후에 뒤에 빨간점이 표시되는 방식이였는데 문제는 타이틀 텍스트가 늘어 났을때 발생했다. 줄넘김을 막기위해 생략기호 처리를 해야 하는데 css로 처리하기 위해서 들어가는 많은 속성 중에 width 값과 overflow: hidden이 들어가야했다. 이것으로 인해 배치되던 타이틀 텍스트 영역 밖의 빨간 포인트 점이 가려지고 width 값을 100% 로 설정하면 텍스트 길이에 상관 없이 항상 맨 끝에 배치된다.

  • 💡 해결방식

    우선 타이틀 텍스트 영역의 padding-inline-end 값으로 오른쪽 공간에 여백을 주어 빨간색 포인트 점이 텍스트와 겹쳐지 않으면서 보일 수 있게 처리하였고, 타이틀 텍스트의 width 값은 fit-content 값을 줘서 유연하게 처리했고 생략 기호 처리를 위해 max-width: 100%를 주는 방식으로 타이틀 텍스트가 길던 짧던 빨간색 포인트 점이 원하는 곳에 배치되도록 유연하게 처리했다.

  • ❓ 문제점

    <dialog> 요소로 처리한 모달창이 메인페이지 진입시 접근성을 위해 포커스를 dialog 영역 안에 가두는 가상 요소인 ::backdrop이 활성화가 안되는 문제가 생겼다.

  • 💡 해결방식

    <dialog> 요소에 open 속성 값을 준 것이 문제였다. js의 .showModal(); 메소드 값을 통해 모달 요소를 활성화 시켜야 ::backdrop이라는 가상 요소가 생성된다는 것을 알게 되어 해당 방식대로 처리했다.
